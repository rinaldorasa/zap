In JavaScript, `NaN` stands for "Not-a-Number." It is a special value that represents the result of an invalid or undefined mathematical operation. When you perform a mathematical operation that doesn't produce a meaningful numeric result, such as dividing zero by zero or trying to parse a non-numeric string as a number, the result is `NaN`.<br>
																																																																																					   <br>
The quirk with `NaN` in JavaScript is that it has some unique characteristics when compared to other values. Here are a few notable points:                                                                                                                                                                                                            <br>
																																																																																					   <br>
1. NaN is considered a numeric value, but it is not equal to any other value, including itself. This means that if you try to compare `NaN` with another `NaN` using the equality operator (`==` or `===`), it will always evaluate to false.                                                                                                          <br>
																																																																																					   <br>
   ```javascript                                                                                                                                                                                                                                                                                                                                       <br>
   console.log(NaN === NaN); // false                                                                                                                                                                                                                                                                                                                  <br>
   console.log(NaN == NaN); // false                                                                                                                                                                                                                                                                                                                   <br>
   ```                                                                                                                                                                                                                                                                                                                                                 <br>
																																																																																					   <br>
2. The `isNaN()` function is commonly used to check if a value is `NaN`. However, it has some quirks as well. It returns `true` if the argument passed to it is `NaN` or if it cannot be converted to a valid number. This means that `isNaN()` might return `true` for values that are not actually `NaN`.                                            <br>
																																																																																					   <br>
   ```javascript                                                                                                                                                                                                                                                                                                                                       <br>
   console.log(isNaN(NaN)); // true                                                                                                                                                                                                                                                                                                                    <br>
   console.log(isNaN("Hello")); // true                                                                                                                                                                                                                                                                                                                <br>
   console.log(isNaN("123")); // false                                                                                                                                                                                                                                                                                                                 <br>
   console.log(isNaN(123)); // false                                                                                                                                                                                                                                                                                                                   <br>
   ```                                                                                                                                                                                                                                                                                                                                                 <br>
																																																																																					   <br>
3. To perform reliable checks for `NaN`, the `Number.isNaN()` method is recommended, as it only returns `true` if the provided value is exactly `NaN`.                                                                                                                                                                                                 <br>
																																																																																					   <br>
   ```javascript                                                                                                                                                                                                                                                                                                                                       <br>
   console.log(Number.isNaN(NaN)); // true                                                                                                                                                                                                                                                                                                             <br>
   console.log(Number.isNaN("Hello")); // false                                                                                                                                                                                                                                                                                                        <br>
   console.log(Number.isNaN("123")); // false                                                                                                                                                                                                                                                                                                          <br>
   console.log(Number.isNaN(123)); // false                                                                                                                                                                                                                                                                                                            <br>
   ```                                                                                                                                                                                                                                                                                                                                                 <br>
																																																																																					   <br>
It's important to handle `NaN` carefully in JavaScript to ensure proper behavior in your code when dealing with numeric calculations or user input.                                                                                                                                                                                                    <br>

Problema:
<br>
La proliferazione di <code>ID</code> comporta una possibile confusione
sia a livello semantico che propriamente a livello di programmazione din
in quanto ID sono univoci e lap presenza di duplicati id nel programma 
genera comportamenti imprevedibili nella esecuzione .

<br><br>

<ol>
<li>
id="CONNESSIONE_SEMANTICA"
<br>

Nodo principale del programma e rappresenta un elemento completamente
allo scoperto nel senso che se venisse comabiato non ci sarebbe 
combinazione tra <i>compilato</i> e <i>sorgente</i>.
<br><br>

Soluzione:<br>
<ol>
<li>
<pre>
&#60;head>
&#60;script>
 arg00="CONNESSIONE_SEMANTICA";
&#60;/script>
&#60;/head>
</pre>
</li>
<li>
<pre>

&#60;section>
&#60;div id="CONNESSIONE_SEMANTICA">?CONNESSIONE_SEMANTICA&#60;/div>
&#60;/section>

</pre>
</li>
<pre>

document.getElementById(arg00).innerHTML

</pre>

</ol>
<br>
Qui il programma rincipal gestisce tramite una variabile
globale il collegamento con l'esterno. Una volta che viene recuperato
il file esterno js rimane indeciso cosa succederà con id se non
viene assegnato un valore alla variabile <code>arg00</code>
<br><br>
Un interessante miglioramento sarebbe combinare una funzione con lancio in argomento <code>arg00</code> di <code>ID</code> quindi utilizzando <i>un unico passaggio</i> .

<br><br>
Una volta sistemato <code>CONNESSIONE_SEMANTICA</code> tutti gli
altri IDs sono di pertinenza del file esterno js

<br>
</li>


<li></li>
</ol>

<br><br>


I file di programma devono essere abbinati in due versioni
<ol>
<li>Tutto il codice in un unico file <code>html</code></li>
<li>Un file html che ripesca dei file esterni <code>js</code></li>

</ol>

<br>

La composizione di questi due programmi rievoca la
struttura <i>file sorgente</i> e <i>file compilato</i> . Infatti
riuscire a creare un ecosistema tra html e ecmascript richiede una
mentalità <i>semantica</i> altrimenti si perde di vista la funzione
del programma.

<br><br>
Con il tempo alcuni miglioramneti (anch estetici) diventano necessari
e occorre in breve tempo saper mettere mano alla parte del codice rigurdante
quella particolare funzionalità.


<br><br>

Nella fase sperimentale del programma si usano denominazioni <i>al volo</i>
collegate ad altre denominazioni al volo come se fossero password tra
un elemento e l'altro. Tuttavia quando il quadro da <b>bottom-up</b> si 
trasforma in <i>ecosistema</i> occorre collegare in modalità <b>logica</b>
il tutto.

<br>
Le denominazioni <i>al volo</i> fatte in questa prospettiva dovrebbero in
modo univoco essere passibili di sostituzione semantica tramite un editor
esterno al programma.
